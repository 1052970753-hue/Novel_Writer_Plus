# Novel Writer Plus

> **AI 小说写作助手 — Claude Code Skill**
> 
> 七阶段创作流程 + 17人Agent团队协作 + 反AI检测 + 卡文突破系统

---

## 项目简介

Novel Writer Plus 是一个专为 [Claude Code](https://claude.ai/code) 设计的 AI 小说创作 Skill。它通过 **七阶段创作流程** 和 **17人Agent团队协作系统**，帮助作者从零开始完成高质量长篇小说创作。

### 核心能力

- **七阶段完整创作流程** — 从世界观构建到终稿润色，覆盖小说创作全生命周期
- **17人Agent团队** — 主编、角色设计师、情节架构师、对话专家、风格校对等各司其职
- **反AI检测策略** — 内置多种写作风格变换技术，提升文本自然度
- **卡文突破系统** — 五步诊断法 + 八种卡文类型解决方案
- **断更重启SOP** — 四级断更分级 + 五步重启法，让每次断更都能无缝回归
- **版本管理系统** — history/文件夹自动归档所有变更，永远可以回溯
- **项目状态卡** — 新会话秒恢复上下文，不再丢失创作进度

## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/1052970753-hue/Novel_Writer_Plus.git

# 复制 skill 到 Claude Code 目录
# macOS / Linux
cp Novel_Writer_Plus/skill.md ~/.claude/skills/Novel_Writer_Plus/skill.md

# Windows
copy Novel_Writer_Plus\skill.md %USERPROFILE%\.claude\skills\Novel_Writer_Plus\skill.md
```

### 使用

在 Claude Code 中输入：

```
/Novel_Writer_Plus
```

然后按照引导，依次完成七阶段创作。

## 七阶段创作流程

| 阶段 | 名称 | 核心任务 | 产出物 |
|:---:|------|---------|--------|
| **1** | **构思与定位** | 确定题材、目标读者、核心卖点、故事内核 | 构思定位文档 |
| **2** | **世界观构建** | 时代背景、力量体系、社会结构、地理环境、术语表 | 世界观文档、术语表 |
| **3** | **角色设计** | 主角、配角、反派的人物档案，含动机、认知边界、关系张力图 | 人物档案全集 |
| **4** | **大纲与架构** | 分卷结构、事件分段、主线/支线规划、伏笔登记、章梗概 | 大纲文档、伏笔登记表 |
| **5** | **正文创作** | Agent团队分工撰写，逐章完成初稿到终稿 | 章节正文文件 |
| **6** | **连载管理** | 读者反馈收集、修改决策、连载节奏把控 | 反馈日志、修改记录 |
| **7** | **精修润色** | 风格校对、去AI化处理、节奏优化、终稿定稿 | 精修版终稿 |

## 17人Agent团队

Novel Writer Plus 采用多Agent协作架构，每个Agent有明确的职责边界：

### 核心创作团队

| Agent | 职责 | 关键能力 |
|-------|------|---------|
| **主编 Agent** | 统筹全局，任务分配，质量把控 | 决策仲裁，资源调度 |
| **角色设计师** | 创建人物档案，维护角色一致性 | 动机分析，认知边界设计 |
| **情节架构师** | 设计主线/支线，管理因果链 | 事件分段，冲突升级 |
| **对话专家** | 撰写高质量对话，确保角色声音辨识度 | 动作节拍，潜台词设计 |
| **场景描写师** | 环境描写，氛围营造 | 感官描写，空间构建 |
| **心理描写师** | 内心独白，情感刻画 | 视角控制，意识流 |

### 质量审核团队

| Agent | 职责 | 关键能力 |
|-------|------|---------|
| **审阅者** | 逻辑校验，一致性检查 | 漏洞发现，因果追溯 |
| **读者视角** | 模拟读者体验，发现阅读障碍 | 信息差管理，悬念评估 |
| **风格校对** | 语言风格一致性，节奏把控 | 句式分析，修辞优化 |
| **细节抠取者** | 专找硬伤，检验合理性 | 逻辑推演，常识校验 |

### 专项辅助团队

| Agent | 职责 | 关键能力 |
|-------|------|---------|
| **去AI优化师** | 消除AI痕迹，提升文本自然度 | 风格变换，困惑度提升 |
| **伏笔管理者** | 登记/跟踪/回收伏笔 | 草蛇灰线，前后呼应 |
| **时间线管理员** | 维护时间线账本，防止时间线矛盾 | 时间逻辑校验 |
| **命名顾问** | 角色/地名/功法命名 | 大俗大雅，名字即命运 |
| **节奏诊断师** | 分析剧情节奏，发现拖沓/过快段落 | 信息密度分析 |
| **冲突设计师** | 设计升级冲突，制造张力 | 三幕结构，高潮设计 |
| **世界观审计员** | 检查设定一致性，防止设定吃书 | 规则追溯，边界校验 |

## 高级功能

### 卡文突破系统

遇到写作瓶颈时，内置 **五步诊断法** 快速定位问题：

```
1. 不知道写什么？ -> 情节卡 -> 回到大纲
2. 知道但写不出？ -> 执行卡 -> 换方法
3. 写得出来但觉得烂？ -> 信心卡 -> 继续写
4. 根本不想写？ -> 倦怠卡 -> 换件事做
5. 反复改不完？ -> 完美主义卡 -> 标TODO跳过
```

覆盖 **8种卡文类型**：情节卡、人物卡、场景卡、对话卡、情绪卡、节奏卡、完美主义卡、倦怠卡，每种都有具体的突破方案。

### 断更重启 SOP

四级断更分级恢复方案：

| 断更时长 | 等级 | 恢复方案 |
|---------|------|---------|
| 1-3天 | 轻度 | 读状态卡 + 上一章最后500字 -> 续写 |
| 4-14天 | 中度 | 读角色声音 + 最近5章 + 500字热身 |
| 15-30天 | 重度 | 全量恢复 + 2000字热身稿 |
| >30天 | 严重 | 全量恢复 + 决策日志 + 大纲审计 |

### 反AI检测策略

内置多种写作风格变换技术：

- **句式多样性**：长短句交替，避免AI常见的均匀分布
- **词汇自然度**：使用口语化表达，避免过于书面化的AI用语
- **节奏变化**：故意制造不规则的节奏模式
- **个人语癖**：为每个角色设计独特的语言习惯
- **标点风格**：变化标点使用模式，避免AI的规范化标点

### 版本管理系统

所有修改自动归档到 `history/` 文件夹：

- **不删只移**：旧版内容完整保存，附带变更记录
- **变更总览**：按时间线汇总所有修改历史
- **回滚路径**：每次变更都记录如何恢复到旧版
- **影响评估**：分析变更对其他章节/人物的影响

## 使用示例

### 创建新小说

```
/Novel_Writer_Plus

用户：我想写一部玄幻修仙小说，主角从凡人开始修炼
```

系统会引导你完成：
1. 确定小说名和核心设定
2. 设计主角和主要配角
3. 构建世界观和力量体系
4. 规划大纲和章节结构
5. 开始正文创作

### 续写已有小说

```
/Novel_Writer_Plus

用户：继续写《XXX》，上次写到第15章
```

系统会自动：
1. 读取 `00-项目状态卡.md` 恢复上下文
2. 检查最近章节和待办事项
3. 准备续写环境

### 解决卡文问题

```
/Novel_Writer_Plus

用户：第8章写不下去了，不知道接下来写什么
```

系统会：
1. 运行五步诊断法
2. 分析卡文类型
3. 提供具体突破方案
4. 协助重新规划章节内容
